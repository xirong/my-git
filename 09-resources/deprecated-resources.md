# 归档资料

这个页面记录仍有历史参考价值，但不建议作为主学习路径的内容。

## 为什么需要归档页

Git、GitHub、GitLab 和团队协作工具都在持续变化。旧文章里的命令、截图、工具推荐、平台功能说明，可能已经不适合直接照做。

归档并不代表内容没有价值，它主要用于：

- 了解历史背景
- 对照旧流程和新流程差异
- 保留迁移线索
- 避免读者把旧做法当成当前推荐方案

## 当前归档候选

| 内容 | 原因 | 推荐阅读 |
| --- | --- | --- |
| `using-svn.md` | SVN 对比仍有学习价值，但不适合作为 Git 主路径 | [Git vs SVN](../01-getting-started/git-vs-svn.md) |
| 旧版 Git 工具链接 | 工具状态可能变化，部分资料需要重新核验 | [Tools](tools.md) |
| 旧版 GitHub 使用说明 | GitHub 功能和界面变化较快 | [GitHub Engineering Governance](../04-github-engineering/github-engineering-governance.md) |
| 旧版工作流教程 | 需要结合团队规模、发布频率和 CI 能力重新判断 | [Team Git Workflow Guide](../03-team-collaboration/team-git-workflow-guide.md) |

## 阅读建议

阅读归档内容时，优先确认三件事：

1. 命令在当前 Git 版本中是否仍然适用
2. 平台功能在当前 GitHub / GitLab 中是否仍然存在
3. 推荐流程是否适合当前团队规模和发布方式
