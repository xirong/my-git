## 内容原则

1. 少堆链接，多讲判断
2. 少列命令，多讲场景
3. 少追求大而全，多给可执行路径
4. 每篇文章都要能回答“什么情况该用，什么情况别用”

## v2.0 Release 叙事

AI 编程正在改变软件开发方式，Git 的价值也随之变化。

v2.0 会围绕团队协作、GitHub 工程治理、AI 代码审查、事故恢复和大仓库实践，系统补齐 Git / GitHub 实战内容。
