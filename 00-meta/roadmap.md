## 内容原则

1. 少堆链接，多讲判断
2. 少列命令，多讲场景
3. 少追求大而全，多给可执行路径
4. 每篇文章都要能回答“什么情况该用，什么情况别用”

## v2.0 Release 叙事

2015 年，这个仓库开始整理 Git 学习资料，帮助中文开发者入门 Git。

2026 年，AI 编程正在改变软件开发方式，Git 的价值也随之变化。

它已经从版本控制工具，扩展成团队协作系统、工程治理底座、AI 代码变更的人类审查账本。

所以 v2.0 会面向现代工程团队与 AI 编程时代，系统补齐 Git / GitHub 实战内容。
